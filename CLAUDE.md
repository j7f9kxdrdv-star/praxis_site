# Praxis Prep — Project Guide

## What is this?
An MCAT prep marketing website for Praxis Prep. Built with Next.js, Tailwind CSS, and TypeScript. Deployed on Vercel.

## Design Direction
- **Aesthetic:** Premium, academic, trustworthy — inspired by Crimson Education
- **Typography:** Serif font for headings (prestige), sans-serif for body (readability)
- **Color palette:** Restrained — deep brand color used sparingly on CTAs/accents, white/off-white backgrounds, dark text
- **Spacing:** Generous whitespace between sections. Nothing cramped.
- **Animations:** Subtle fade-ins only. No flashy effects.

## Content Direction
- **Voice:** Aspirational but accessible. Confident but not arrogant. Speaks to pre-med students directly.
- **Modeled after:** Blueprint Prep MCAT (value props, social proof, course tiers) — but NOT their guarantee-based messaging
- **Key themes:** Cost-effectiveness, unbelievable customer experience, being at the student's side through the process, personal mentorship
- **NEVER guarantee specific scores or point increases.** Testimonials showing real student results are fine (that's social proof), but the brand does not promise outcomes that depend on the student's ability to perform.
- **Differentiator:** The relationship and experience, not a number. "We give you the tools — and we're right there beside you."

## Tech Stack
- Next.js 16 (App Router)
- Tailwind CSS 4
- TypeScript
- Deployed on Vercel, code on GitHub

## Design Rules
- **Mobile-first is mandatory.** Every component, layout, and page must be designed and built for mobile first, then scaled up for tablet/desktop. Always verify mobile rendering before considering a task complete.

## Conventions
- All components live in `/components`
- Page routes use the App Router (`/app` directory)
- Use Tailwind utility classes for styling (no separate CSS files beyond globals.css)
- Keep components modular — one section per file
